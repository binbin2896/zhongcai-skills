# 技能复盘与持续优化机制

**说明：** 技能体系不是一次性建成的，需要通过定期复盘不断迭代。本机制设计了三级复盘节奏，确保知识库和模板持续优化。

---

## 一、复盘节奏

```
┌─────────────────────────────────────────────┐
│             复盘节奏（三级）                  │
├─────────────────────────────────────────────┤
│                                             │
│  每案复盘（案件审结后）                      │
│  ┌─────────────────────────────────────┐    │
│  │ 时间：裁决书送达后3日内              │    │
│  │ 时长：15分钟                         │    │
│  │ 动作：快速填写案件复盘卡              │    │
│  └─────────────────────────────────────┘    │
│              ↓                               │
│  月度复盘（每月月底）                        │
│  ┌─────────────────────────────────────┐    │
│  │ 时间：每月最后一周                   │    │
│  │ 时长：30分钟                         │    │
│  │ 动作：汇总本月案件复盘卡，提炼规律    │    │
│  └─────────────────────────────────────┘    │
│              ↓                               │
│  季度复盘（每季度初）                        │
│  ┌─────────────────────────────────────┐    │
│  │ 时间：每季度第一个周末               │    │
│  │ 时长：60分钟                         │    │
│  │ 动作：能力自评+知识库更新+模板迭代    │    │
│  └─────────────────────────────────────┘    │
│                                             │
└─────────────────────────────────────────────┘
```

## 五、技能迭代触发规则

以下情况触发**立即更新**而非等待复盘节点：

| 触发条件 | 更新对象 | 操作 |
|---------|---------|------|
| 新司法解释/法规出台 | 对应裁判要点库 | 7日内更新并标注版本 |
| 收到法院撤销/不予执行裁定 | 司法审查手册 | 分析原因，更新风险点 |
| 发现模板关键字段缺失（使用中） | 对应模板 | 即时补充并版本号+1 |
| 新培训 | 案例库+模板 | 提取新案例，比对更新 |
| 月度复盘中同一问题出现≥3次 | 对应模板/清单 | 立即迭代，下月底验证效果 |

## 六、版本管理规范

所有模板和知识库文件采用以下命名规则：

```
[序号]-[名称]-v[主版本].[副版本].md
```

- **主版本**：结构性或关键内容变更
- **副版本**：小幅修正、格式调整、案例补充

每次更新在文件头部记录变更日志：

```
---
version: 1.0
updated: 2026-06-01
change: 初始版本
---
```

## 七、年度体检

每年12月底做一次全面体检：

| 项目 | 内容 | 完成 |
|------|------|------|
| 能力自评 | 年度对比9个维度变化趋势 | □ |
| 模板盘点 | 所有模板的使用率统计 | □ |
| 知识库审计 | 裁判要点库的准确性和时效性 | □ |
| 案例库盘点 | 年内新增案例数量和价值评估 | □ |
| 目标达成 | 年初设定目标的完成情况 | □ |
| 明年规划 | 下年度的重点方向 | □ |

---

> **核心理念：** 复盘不是为了"检查工作"，而是为了让经验不被下一次案件浪费。每个案件审结后，留下一点东西，一年后你会感谢自己。
