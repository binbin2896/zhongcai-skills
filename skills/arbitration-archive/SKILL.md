# 案归.skill · 案件复盘与经验入库

> "案"=案件，"归"=归拢归档。每个案件留下一点东西，一年后你会感谢自己。

## Overview

本Skill在案件审结后执行复盘归档。核心理念：复盘不是为了"检查工作"，而是为了让经验不被下一次案件浪费。通过每案复盘卡→月度提炼→季度迭代的三级机制，持续积累裁判经验。

核心定位：🔵 AI填充复盘卡、提取关键数据 → 🟢 AI识别可入库的新案例、标注模板改进点 → 🔴 人确认关键经验、决定知识库更新

## When to Use

| 触发信号 | 场景 |
|---------|------|
| "帮我把这个案子归档" | 裁决书已送达，需要结案整理 |
| "复盘一下这个案件" | 总结经验教训 |
| "这个案子的经验可以入库吗" | 评估新案例价值 |

不适用场景：案件尚在审理中 / 未出具裁决书

## AI/Human 分工

🔵 AI自动：填充案件基本信息、汇总模板使用情况、生成数据统计
🟢 AI建议：标出值得入库的案例、建议模板改进点、识别高频问题
🔴 人决定：确认关键经验、批准案例入库、决定知识库更新

## Execution Flow

### Step 1：加载案件信息

从裁判文书和审理过程中提取：案号、案由、结案方式、开庭次数、审限情况、模板使用情况。

### Step 2：填写每案复盘卡

```🟦AI填充 🟩人确认
═══════════════════════════════════════════
案件复盘卡
═══════════════════════════════════════════

📋 [案号] [案由] | [裁决/调解/撤案] | [审限内/超审限]
     🟦自动        🟦自动         🟦自动        🟦自动

🟢 本案亮点（经验要固化）
🟦AI：识别裁判中的关键判断——提炼法律依据+裁判逻辑+方法论
🟩人：确认AI提炼是否准确 → 补充"下次怎么做"的行动指南
[AI建议 + 人确认]

🔴 最大挑战（教训要记录）
🟦AI：识别审理中最棘手的环节——事实查明/法律适用/程序合规等
🟩人：确认AI定性是否准确 → 写出"下次如何处理"的替代方案
[AI建议 + 人确认]

🟡 模板使用反馈
🟦AI：根据审理过程推断哪些模板被使用、效果如何
🟩人：修正AI推断，补充未使用的模板及原因
- 阅卷笔录：□ 用了 □ 没用 → 效果：[好/一般]
- 庭审清单：□ 用了 □ 没用 → 效果：[好/一般]
- 裁决书模板：□ 用了 □ 没用 → 效果：[好/一般]
- 校核清单：□ 用了 □ 没用 → 效果：[好/一般]

| 💡 知识库更新建议
🟦AI：根据本案裁判要旨和入库标准，自动生成以下建议
🟩人：逐项确认后打勾或否决
- [ ] 建议作为新案例入库（理由：）
- [ ] 模板需要改进（内容：）
- [ ] 法律依据需要更新（内容：）
```

#### 🔴 CHECKPOINT 🛑 Step2复盘卡确认
AI已填充复盘卡初稿。请逐项确认：亮点是否提炼为可复用经验？挑战是否写出了"下次怎么做"？使用反馈是否准确？确认后回复"复盘卡OK"进入入库评估。

### Step 3：评估案例入库价值

| 入库标准 | 判断 |
|---------|------|
| 涉及新颖法律问题 | □ 是 □ 否 |
| 裁判思路有参考价值 | □ 是 □ 否 |
| 与现有案例不重复 | □ 是 □ 否 |

#### 🔴 CHECKPOINT 🛑 Step3入库价值确认
AI已根据三项标准给出入库建议。请判断：此案是否值得入库？如"是"，AI将同时生成入库内容草稿供确认；如"否"或"待定"，则跳过入库流程。回复"入库/否/待定"。

### Step 4：触发机制检查
- 发现模板关键字段缺失 → 标记模板版本升级建议
- 新司法解释/法规影响裁判要点 → 标注知识库更新
- 月度复盘中同一问题出现≥3次 → 标记为必改项

### Step 5：输出归档报告

```🟦AI生成 🟩人确认
═══════════════════════════════════════════
归档报告
═══════════════════════════════════════════

📌 [案号] [案由]          🟦自动
🏁 结案方式：[裁决/调解/撤案]  🟦自动
📊 审限：[__天] 超审限：[是/否]  🟦自动

📝 关键经验
🟦AI：汇总复盘卡亮点和挑战，提炼核心方法论
🟩人：确认后精炼为可复用的3-5句话
[人确认后的经验总结]

📚 建议入库案例：[是/否]  🟦AI建议 🟩人确认
📋 模板改进建议：[__项]   🟦AI统计 🟩人确认每项
⚡ 需立即更新：[是/否 — 触发规则]  🟦自动判断

📅 下阶段：月度汇总/季度复盘
```

#### 🔴 CHECKPOINT 🛑 Step5出口确认
执行者确认归档报告内容准确、入库建议合理、模板改进已记录后再关闭。如有"待定"项，则在报告开头标注⚠️待处理。同意请回复"确认归档"。

## 反例与参考（Worked Example）

### 完整示例：建工案件归档复盘

**输入：** 通仲裁字第2025-0189号建设工程施工合同纠纷，裁决书已送达。申请人请求支付工程款182万元，仲裁庭支持136万元。争议焦点为"项目经理签字确认的工程量清单是否构成结算依据"。

**❌ 坏例（归档流于形式）：**
```
🟢 本案亮点：仲裁庭认定工程量清单有效
🔴 最大挑战：申请人对鉴定结论有异议
📚 建议入库：是
```
→ 问题：亮点没有写出"为什么有效"的判断标准，挑战没有写出"如何处理异议"的方法论，入库建议无理由。

**✅ 好例（真正提炼经验）：**
```
🟢 本案亮点：项目经理签字确认的工程量清单，虽然合同约定"须加盖公章方为有效"，但根据《民法典》第172条表见代理原理，项目经理日常进度确认行为已构成职务代理。下次遇到类似情况，重点审查：①签字人是否在项目现场常驻；②此前是否一直以该方式确认过（交易习惯）；③相对人是否善意。
🔴 最大挑战：申请人对鉴定结论有异议但不申请重新鉴定——仲裁庭依职权要求鉴定人出庭接受质询，异议自动消解。经验：鉴定异议应先引导当事人申请鉴定人出庭，而非直接启动重新鉴定程序。
📚 建议入库：是——理由：项目经理签字效力认定涉及表见代理在新《公司法》施行后的适用边界，裁判思路有普遍参考价值。
```

**📊 对比要点：**
| 对比项 | 坏例 | 好例 |
|--------|------|------|
| 亮点 | 事实复述（"认定有效"） | 法律依据+判断标准+下次行动指南 |
| 挑战 | 现象描述（"有异议"） | 处理过程+替代方案+经验总结 |
| 入库建议 | 无理由 | 明确的价值定位和入库依据 |
| 可复用性 | 低（仅限于知这件事发生过） | 高（下次遇到类似案件可以直接套用思路） |

## 失败模式编码（Failure Mode Encoding）

| 故障码 | 症状 | 触发条件 | 一线修复 | 仍失败兜底 |
|--------|------|---------|---------|-----------|
| F001 | **案件信息不完整** | Step1加载时缺少案号/案由/结案方式等必要字段 | 提示用户补全缺失字段（具体列出缺哪些），暂存已填部分 | 转为"待归档"草稿状态，在复盘卡上方标注⚠️信息不完整标识 |
| F002 | **复盘卡填写卡顿** | Step2中模板字段太多，用户跳过大部分只填"已归档"便结束 | AI主动识别未填字段，逐一提问引导："本案的最大挑战是什么？(可写'无重大挑战')" | 退化为快速复盘模式：只填亮点+挑战+入库建议三项必填，其余自动标记"待补充" |
| F003 | **入库价值判断困难** | Step3评估时三项入库标准都模糊（"好像有参考价值但不确定"） | AI展示现有案例库中5个最相似案例的入库依据，帮助用户对标判断 | 标记为"待定-月度复审再评"，28天后自动推送提醒要求完成入库评估 |
| F004 | **月度复盘遗忘** | 用户完成归档后未触发月度/季度复审流程，三级机制中断 | 归档报告输出时自动创建30天后日历提醒，并写入复盘卡头部的"复审倒计时" | 每周推送一次"待复审案件清单"（含最近30天内未复审的归档报告链接） |
| F005 | **知识库写入冲突** | Step3判定"建议入库"，但知识库中已存在相同案由/相似事实的案件 | AI自动对比冲突案例的争议焦点和裁判要旨，用diff形式展示差异 | 标记为"冲突-待人工确认"，将冲突比较结果写入备注字段后提交人审 |

## 反模式与黑名单（Anti-Patterns）

| # | 反模式 | 为什么有害 | 正确做法 |
|---|--------|-----------|---------|
| 1 | **归档流于形式**：复盘卡填了但经验没真正提炼，仅仅在"亮点"栏写"无"或在"挑战"栏复述案情 | 浪费复盘机会，下次遇到同类问题仍然从头摸索，三级机制变成三级形式主义 | 每条亮点和教训必须写出"下次遇到类似情况可以怎么做"，写成可执行的行动建议而非事件描述 |
| 2 | **模板不迭代**：复盘发现了模板字段缺失但只改本次复盘卡，未更新模板本身和版本号 | 模板永远停留在v1.0，同一个问题下个案件再犯一遍——复盘的最大价值在于反哺工具，而非仅记录 | 模板正文内容改变后必须版本号+1，并记录 changelog。即使改动只有一行，也要升级 |
| 3 | **案例入库标准过松**：每个案件都入库，入库标准形同虚设 | "入库"变成"所有结案案件"，知识库被低价值案例淹没，入库反而降低了检索效率 | 严格遵从3项入库标准：新颖法律问题 + 裁判思路有参考价值 + 与现有案例不重复，三项缺一不可 |
| 4 | **复盘即终结**：归档报告写完即束之高阁，月度/季度复审从未执行 | 三级机制的第二层和第三层从未触发，经验停留在"读完就忘"的文档层面，不能转化为裁判能力的实际提升 | 归档报告输出时自动设置一个30天后的日历提醒（月度复审），复审完成后再设置90天周期提醒 |
| 5 | **AI结论照单全收**：AI提取的关键经验不经验证就写入知识库 | AI可能遗漏案件的特殊背景或误判裁判思路的价值，导致知识库出现误导性案例标签 | 所有AI建议的"案例入库"和"模板改进"必须经人确认，且确认标记要写入归档报告状态栏 |

## 自洽性与质量自检（Verification & Self-Check）

### 输出前自检表（5维）

| # | 检查项 | 标准 | 不通过则 |
|---|--------|------|---------|
| 1 | **复盘卡完整性** | 亮点/挑战/模板反馈/入库建议四项都填写，且亮点和挑战包含"下次怎么做" | 退回Step2补充，标记"未完成" |
| 2 | **经验可复用性** | 每条经验不是事实复述（"认定有效"）而是行动指南（"下次审查哪些材料"） | AI标注问题字段，提示改写 |
| 3 | **入库理由充分性** | 判定"入库"的必须有明确理由，至少满足3项入库标准中的1项 | 改为"待定"，由月度复审决定 |
| 4 | **触发链完整性** | 模板改进建议已记录版本号（v1.x→v1.x+1）、知识库更新已标注触发规则 | 退回Step4补全触发动作 |
| 5 | **复审倒计时已设置** | 归档报告中包含30天后的复审提醒，日历提醒已创建或已记录在案 | 自动创建提醒，或写入归档报告头部"⚠️需手动设置复审" |

### 🛑 强制回退规则

| 条件 | 动作 |
|------|------|
| 自检表中任意一项为"否" | 退回对应的执行步骤修复，不通过则归档报告标记为"草稿"而非"已完成" |
| 自检表连续2轮未通过 | 降级为"快速归档"模式（仅填写三项必填字段），标记为⚠️需要人工复核 |
| AI建议的入库案例被用户否决（连续3次） | AI的入库建议阈值自动调高20%，减少误判 |
