# 裁笔.skill · 裁决书草拟辅助

> "裁"=裁断，"笔"=书写。九段为骨，案例为鉴，说理为魂。

## Overview

本Skill在庭审结束后辅助起草裁决书。提供四种范本（简式/普通/复杂/调解）、匹配相似案例、按"四阶层"框架组织裁判说理。

核心定位：🔵 AI搭建结构、填充程序信息、匹配案例 → 🟢 AI提供说理范本和风险提示 → 🔴 人完成事实认定、法律论证、作出裁决

## When to Use

| 触发信号 | 场景 |
|---------|------|
| "帮我起草裁决书" | 庭审结束，需要撰写裁决书 |
| "这个案子裁决书怎么写" | 需要结构指导和案例参考 |
| 裁笔.skill 被 经纬.skill 链式调用 | meta-skill编排时自动触发 |

不适用场景：裁决书已完成草稿（应走 校核.skill）/ 未开庭（应先走 问庭.skill）

## AI/Human 分工

🔵 AI自动：选择模板版本、填充程序信息、生成时间轴、匹配相似案例
🟢 AI建议：提供说理范本、标注需要论证的焦点、提示引用法律
🔴 人决定：事实认定、法律适用、责任划分、裁决主文

## Execution Flow

### Step 1：加载案件信息

从阅卷笔录、庭审记录、证据材料中提取关键信息。

🔴 CHECKPOINT 🛑 — 输入确认：展示提取的案件概况给用户确认。确认→进入Step 2；缺信息→引导补充（"是否有庭审笔录？争议焦点是否已确认？"）；未开庭→触发F001降级。

### Step 2：选择裁决书版本

| 案件类型 | 推荐模板 | 说明 |
|---------|---------|------|
| 事实清楚、争议不大 | 简式裁决书 | 简易程序，说理可简化 |
| 标准案件 | 普通程序裁决书 | 标准九段式，完整说理 |
| 建工/群体性/复杂 | 复杂案件裁决书 | 增补时间轴+四阶层+专业术语 |
| 调解成功 | 调解/和解撤案裁决书 | 确认协议效力 |

🔴 CHECKPOINT 🛑 — 版本确认：展示所选版本给用户确认。确认→进入Step 3；换版本→按指令切换；不确定→简要说明各版本的适用场景后让用户选。

### Step 3：搭建九段式结构

```
1. 当事人与代理人信息
2. 案件受理与仲裁庭组成
3. 仲裁请求、反请求与答辩意见
4. 证据采信与事实认定 ← 核心难点
5. 仲裁庭意见（争议焦点逐项分析） ← 核心难点
6. 裁决主文
7. 仲裁费用
8. 裁决履行告知
9. 仲裁员签字与日期
```

### Step 4：匹配相似案例

从案例库中匹配与当前案件争议焦点相似的案例，提供"原稿→问题→修改后稿"的对照参考。

### Step 5：组织说理（四阶层）

```
第一阶层：法律关系定性
第二阶层：请求权基础分析
第三阶层：责任划分
第四阶层：损失计算
```

🔴 CHECKPOINT 🛑 — 定稿确认：展示初稿框架给用户，确认后再输出。

```
═══════════════════════════════════════════
裁决书初稿
═══════════════════════════════════════════

📋 [案号]

一、当事人信息：[AI已填充，请人确认]
二、受理与组庭：[AI已填充]
三、请求与答辩：[AI已归纳，请人确认]
四、证据与事实：[待人完成]
五、仲裁庭意见：[【四阶层框架已搭建】待人完成]
六、裁决主文：[待人完成]
...

📎 参考案例：[案例X] — 匹配争议焦点 [XX]
📎 说理提示：[第X阶层需重点论证]
```

## 生态集成：上下游对接

### 在经纬skill链中的位置

```
问庭.skill (上游) ──→ 裁笔.skill ──→ 校核.skill → 防讼.skill (下游)
     │                     │                     │
     输出：庭审问题清单      输出：裁决书初稿       输出：质检/风控报告
     （含争议焦点确认）     （待校核+防讼）         （通过后方可签发）
```

### 上游对接（问庭.skill → 裁笔.skill）

| 上游产出 | 本skill消费方式 | 传递要求 |
|---------|---------------|---------|
| 争议焦点归纳 | 直接映射到Step 5仲裁庭意见的框架 | 必须有，否则触发F002信息不足 |
| 庭审笔录/关键事实 | Step 4证据认定和Step 5说理的依据 | 建议有，无则输出框架留空待人填 |
| 调解结果 | Step 2裁决书版本选择（调解成功则用调解版） | 可选 |

### 下游对接（裁笔.skill → 校核.skill + 防讼.skill）

| 本skill产出 | 下游消费方式 | 建议 |
|------------|-------------|------|
| 裁决书初稿 | 校核.skill 跑三十项清单 | 草稿完成后必须走校核 |
| 裁决书初稿 | 防讼.skill 做撤裁风险预演 | 出裁决前必须走防讼 |

### 与其他skill的边界

| 不要用裁笔.skill的场景 | 应用 |
|----------------------|------|
| 还没开庭 | 问庭.skill（先出庭审问题清单） |
| 裁决书写完要质检 | 校核.skill |
| 想预判撤裁风险 | 防讼.skill |

## 反模式与黑名单

| # | 反模式 | 为什么不行 | 正确做法 |
|---|--------|-----------|---------|
| 1 | 模板套用忽略个案特殊性 | 建工案件的"黑白合同""背靠背条款"等特殊问题不能简化处理，照模板填会遗漏关键争点 | 先按模板搭框架，再逐段对照案件特殊事实做增补，不能依赖模板直接输出 |
| 2 | 说理与事实脱节 | 法律适用必须与认定的事实相关，引用法条但事实未查清等于空中楼阁 | 四阶层的第一阶层（法律关系定性）必须以已认定的事实为前提，不跳步 |
| 3 | 遗漏被申请人答辩意见 | 被申请人的每一项抗辩都应在仲裁庭意见中回应，否则可能构成程序瑕疵 | 裁决书草拟完后逐项对照被申请人答辩意见，确保0遗漏 |
| 4 | 双务合同解除只判退款不问返还 | 违反《九民纪要》第36条释明义务，只处理了退款没处理车辆/货物返还，导致利益失衡 | 涉及双务合同解除的，必须同时处理双方返还义务，裁决主文中明确"同时履行" |
| 5 | 写完了直接签不发校核 | 跳过三十项校核和司法审查风险评估，直接签发 | 草稿完成后必须先走 校核.skill 再过 防讼.skill，两道关过了再签字 |

## 异常处理与降级路径

| 故障码 | 异常场景 | 触发条件 | 一线修复 | 仍失败兜底 |
|-------|---------|---------|---------|-----------|
| F001 | 用户未开庭就要求起草裁决书 | 用户说"先写写看"，但庭审还没开 | 退回推荐：应先走 问庭.skill 完成庭审 | 用户坚持→标注"未经庭审，裁决书为模拟草稿" |
| F002 | 案件信息不足，无法确定模板版本 | 用户只提供了案由，没有庭审记录和证据材料 | 追问关键信息：是否已开庭？是否有庭审笔录？争议焦点是否已明确？ | 仍不提供→使用普通程序通用模板，标注"信息不全" |
| F003 | 无法匹配相似案例 | 争议焦点类型不在8个案例索引覆盖范围内 | 从参考库中扩大匹配范围（按法条/案由/合同类型匹配） | 仍不匹配→省略案例引用，仅提供说理范本 |
| F004 | 用户对初稿结构不满意 | Step 6 预览时说"这个结构不对" | 按用户要求调整模板版本（简式↔普通↔复杂） | 用户说不清→输出所有版本供选择 |
| F005 | 用户中途中断 | 任意环节用户说"算了，我自己写" | 保存当前进度（已搭框架+已匹配案例），告知后续可继续 | — |

## 使用示例

### 输入

> 案由：建设工程施工合同纠纷。合同为固定总价，已竣工验收。申请人主张被申请人拖欠工程款1250万元及利息，被申请人以工程质量不合格反诉要求赔偿修复费用。庭审已结束，争议焦点：①已完工程量如何认定 ②质量缺陷责任归属 ③优先受偿权是否过期。双方调解不成。

### ❌ 坏例（未使用本skill）

直接输出一段概述性文字，没有分段、没有结构：

```
本案系建工纠纷，申请人要1250万，被申请人说质量有问题。仲裁庭认为工程已验收合格，应支付工程款，但质量缺陷也要处理。裁决被申请人支付1250万。
```

**问题：** 没有九段式结构、没有逐项分析争议焦点、没有四阶层说理、没有引用法律依据、无法看出哪些是AI填的哪些是人写的。

### ✅ 好例（使用本skill）

按九段式组织、四阶层说理、AI/人分工标记清晰（见上文初稿输出）。

### 对比要点

| 维度 | 坏例 | 好例 |
|------|------|------|
| 结构 | 一段概述 | 九段式完整框架 |
| 说理 | 无 | 四阶层逐层深入（定性→请求权→责任→损失） |
| 争议焦点 | 混在一起 | 三个焦点逐一分析 |
| AI/人分工 | 无法区分 | 🔵🟢🔴 标记清晰 |
| 法律引用 | 无 | 民法典第788/799/801/807条等 |
| 案例参考 | 无 | 匹配案例三/五/八 |

## 自洽性与质量自检

### 输出前自检表

| 检查项 | 问题 | 通过标准 |
|--------|------|---------|
| 🎯 核 | 框架是否覆盖了全部争议焦点？ | 每个争议焦点在仲裁庭意见中都有对应段落 |
| 📋 全 | 九段式各段是否完整？ | 九段齐全，无缺段 |
| 🔗 序 | 顺序是否符合九段式规范？ | 证据认定→仲裁庭意见→裁决主文的逻辑链不颠倒 |
| ⚠️ 疑 | 被申请人答辩意见是否全部回应？ | 逐项对照答辩意见，无遗漏 |
| ✅ 用 | 裁决主文是否可执行？ | 内容完整、用词准确、无歧义，确保可强制执行 |

### 🛑 强制回退规则

- **任一❌ → 退回对应步骤补齐**
- **连续2次❌同一检查项** → 建议先走 阅卷.skill 补案件梳理后再来
- **CHECKPOINT 用户中止** → 保存当前进度，按用户方向重走
- **全部✅ → 建议走 校核.skill + 防讼.skill 两道质检后再签发**
