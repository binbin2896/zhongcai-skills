# 仲裁-校核 · 裁决书质量校核与风险排查

> 裁决书出具前的最后一道防线。AI做全面扫描，人做关键确认。

## Overview

本Skill在裁决书草拟完成后执行系统化校核，覆盖六大维度30项检查 + 程序合规10类风险 + 司法审查5大事由。AI逐项扫描并标注风险等级，仲裁员确认每项判断，最终输出校核报告。

核心定位：🔵 AI自动扫描 → 🟢 AI标注建议 → 🔴 人最终确认

## When to Use

触发信号：帮我校核一下裁决书 / 看看有没有撤裁风险 / 走一遍校核流程

不适用：裁决书尚未完成草稿 / 需要AI直接修改裁决书内容 / 案件事实认定争议

## AI/Human 分工

🔵 AI自动执行：逐项过30项清单，匹配风险等级，标出疑点段落
🟢 AI建议：对每个疑点给出判定建议，供人确认
🔴 人决定：确认每个风险项是否属实，决定是否修改裁决书

## Execution Flow

### Step 1：加载裁决书

从当前对话或Feishu文档中获取裁决书草稿。如未提供，提示用户粘贴或提供文档链接。

🔴 CHECKPOINT 🛑 — 裁决书确认：展示裁决书摘要给用户确认。确认→进入Step 2；有更新→等待更新后再加载；不是终稿→标注版本。

### Step 2：三十项校核（六大类逐项扫描）

按六大类逐项扫描，每项输出：

| # | 校核项 | AI判断 | 置信度 | 建议动作 |
|---|--------|--------|--------|---------|
| 1 | 裁决书文号与案号一致？ | 🟢 一致 | 高 | 无需处理 |
| 6 | 案件受理时间、依据是否写明？ | 🔴 缺失 | 高 | 需补充 |
| 24 | 争议焦点归纳是否准确？ | 🟡 范围偏大 | 中 | 建议缩小 |

🔴 CHECKPOINT 🛑 — 校核结果确认：展示🔴项给用户确认。用户确认→进入Step 3；用户调整→按意见修正。

### Step 3：程序合规检查（10类高频风险）

检查10类风险项，标注风险等级和AI扫描结果。

### Step 4：司法审查风险评估（5大事由）

逐项评估违反法定程序、超裁、隐瞒证据、伪造证据、枉法裁决的风险等级。

### Step 5：输出校核报告

🔴 CHECKPOINT 🛑 — 报告确认：展示完整报告给用户，确认后再输出。确认→输出；需调整→按意见修改。

### 填写规范

| 字段 | 填写人 | 规则 |
|------|--------|------|
| 🔴项 | 🔵 AI扫描 + 🟢 AI建议 + 🔴 人确认 | AI标注风险等级后，人必须确认；人与AI判断不一致时以人为准 |
| 🟡项 | 🔵 AI扫描 + 🟢 AI建议 | AI给出建议后，人评估是否需处理 |
| 🟢项 | 🔵 AI扫描 | AI确认通过，人可抽样复核 |
| 司法审查评级 | 🔵 AI评估 + 🔴 人确认 | 综合5大事由风险后给出，人最终定级 |
| 核心建议 | 🟢 AI建议 | AI根据风险项集中度提炼 |

## 使用示例

### 输入

> 裁决书草稿：案由建设工程施工合同纠纷。申请人主张工程款1250万元，被申请人以质量不合格反诉。已竣工验收。争议焦点：①工程量认定 ②质量缺陷责任 ③优先受偿权。

### ❌ 坏例（未使用本skill）

直接输出笼统意见：

```
这个裁决书写得还行，但要注意工程质量问题要写清楚。
```

**问题：** 没有逐项校核、没有🔴🟡🟢分级、没有程序合规和司法审查、无法知道具体哪个环节有问题。

### ✅ 好例（使用本skill）

逐项扫描30项 + 10类合规 + 5大事由，输出分级报告（见Output Template示例）。

### 对比要点

| 维度 | 坏例 | 好例 |
|------|------|------|
| 覆盖面 | 笼统一句 | 30项+合规+审查三层覆盖 |
| 分级 | 无 | 🔴必改/🟡建议/🟢通过 |
| 可操作性 | 不知道改什么 | 每条标注问题+建议+涉及段落 |
| 风险量化 | 无 | 司法审查风险评级[低/中/高] |

## 反模式与黑名单

| # | 反模式 | 为什么不行 | 正确做法 |
|---|--------|-----------|---------|
| 1 | AI误判风险等级 | AI可能将格式问题标为🔴而遗漏实体风险，或把一个低风险标为🔴高 | 先让AI跑全量扫描，再逐项人工复核。AI标注仅供参考，最终风险等级由人定 |
| 2 | 忽略程序合规检查 | 只做了三十项校核就完了，跳过程序合规和司法审查 | 三十项+程序合规+司法审查三者缺一不可，设为Step 2→3→4固定流程 |
| 3 | 过度依赖AI判断 | AI说通过就通过了，人没复核就直接提交核阅 | 🔴项必须人看过再决定，AI的建议标记为"建议"不可替代人 |
| 4 | 校核和防讼混用 | 用校核替代防讼，或反过来 | 校核查"这份裁决书写得好不好"，防讼查"对方会从哪里攻击"，先校核后防讼 |
| 5 | 校核流于形式 | 30项全勾🟢但实际没仔细看，为了走流程而走流程 | 每项校核时必须有依据（裁决书原文引用+AI判断理由），不能空勾 |

## 异常处理与降级路径

| 故障码 | 异常场景 | 触发条件 | 一线修复 | 仍失败兜底 |
|-------|---------|---------|---------|-----------|
| F001 | 裁决书草稿未提供 | 用户说"帮我校核"但没给裁决书 | 要求提供裁决书文本或Feishu链接 | 用户无法提供→终止校核，标注"无输入" |
| F002 | 裁决书不是最终草稿 | 用户给的是初稿或还在修改中 | 标注"初稿校核仅供参考，终稿后需重新校核" | 用户确认后继续，报告标注版本 |
| F003 | 用户不认同AI的风险标注 | 用户说"这个不是🔴，只是🟡" | 按用户意见修正（人的判断优先） | 保留AI标注+用户标注双记录 |
| F004 | 用户对报告格式不满意 | 用户说"太多了/太少了/结构不对" | 按用户要求调整（提供简要版/完整版） | 输出标准版，标注"可自定义" |
| F005 | 用户中途说"不用校核了" | 任意环节用户中断 | 保存已完成的校核项和未完成项清单 | — |

## 自洽性与质量自检

### 输出前自检表

| 检查项 | 问题 | 通过标准 |
|--------|------|---------|
| 🎯 核 | 三十项校核是否全部完成？ | 30项均有结果（🔴/🟡/🟢），无缺项 |
| 📋 全 | 三层检查是否覆盖？ | 三十项+程序合规+司法审查，三层均有输出 |
| ✅ 精 | 🔴项是否全部标注了修改建议？ | 每条🔴都有"问题描述+修改建议+涉及段落" |
| ⚠️ 疑 | 人与AI判断不一致处是否已标注？ | AI标注🟢人认为🔴的，已按人意见修正并记录 |
| 🔗 序 | 报告结构是否完整？ | 分级+风险评级+核心建议，三者齐全 |

### 🛑 强制回退规则

- **任一❌ → 退回对应步骤补齐**
- **连续2次❌同一检查项** → 建议先走 裁笔.skill 完成裁决书后再来校核
- **CHECKPOINT 用户中止** → 保存当前进度，按用户方向重走
- **全部✅ → 输出校核报告，建议再走 防讼.skill 做撤裁风险预演**
