# 仲裁技能体系架构总览

> 本文档是经纬.skill的配套说明，记录仲裁skill链的完整架构和子skill定位。

## 核心设计原则

1. **半自动化**：AI做格式化/匹配/标注，人做事实认定/法律适用/最终判断
2. **渐进式创建**：优先创作ROI最高的子skill（校核→阅卷），其余根据使用反馈逐步补充
3. **引用不内嵌**：模板内容保持为Feishu文档，skill只引用不内嵌，降低维护成本

## Skill 命名与典籍出处

| Skill | ASCII名 | 中文名 | 出处 |
|-------|---------|--------|------|
| Meta | arbitration-master | 经纬.skill | 《左传》"经纬天地曰文" |
| 子1 | arbitration-reading | 阅卷.skill | 行业通用 |
| 子2 | — | 问庭.skill | "问"=庭审提问 |
| 子3 | — | 裁笔.skill | "裁"=裁断，"笔"=书写 |
| 子4 | arbitration-review | 校核.skill | "校"=校勘，"核"=核实 |
| 子5 | — | 案归.skill | "案"=案件，"归"=归档 |

## Skill 系列使用路径图

```
收案 → 经纬.skill（判断阶段）
         ├─ 庭前 → 阅卷.skill → 问庭.skill
         ├─ 庭后 → 裁笔.skill
         ├─ 裁决前 → 校核.skill
         └─ 结案 → 案归.skill
```
